Netlify 404 "Page Not Found" — Fix It

Follow this checklist to diagnose and resolve the most common causes of 404 errors on Netlify.

1 Fix Client-Side Routing React / Vue / Vite

If your homepage loads but refreshing or visiting a direct link like /about shows a 404, Netlify is looking for a real file at that path instead of letting your app handle routing.

Solution: Add a _redirects file to your project's public folder with this rule:

/*    /index.html   200

Commit the file and push to trigger a new deployment. Alternatively, add the equivalent rule in netlify.toml:

[[redirects]]
  from = "/*"
  to   = "/index.html"
  status = 200

2 Verify Your Publish Directory

If even your main URL returns 404, Netlify is likely deploying an empty folder or can't find index.html. Check Site configuration → Build & deploy → Build settings and make sure the publish directory matches your framework's output:

  • Vite: dist
  • Create React App: build
  • Next.js (static export): out
  • Plain HTML/CSS/JS: Leave blank if index.html is at the repo root

3 Use Strict Lowercase File Names

Netlify's Linux servers are case-sensitive. Your entry point must be named exactly index.html — not Index.html or INDEX.HTML — or Netlify won't detect it.

4 Inspect Your Deployment Archive

Still stuck? Download the deploy archive from your Netlify Dashboard (the download icon next to the deployment timestamp), unzip it locally, and confirm that index.html is present in the root of the archive. If it's missing, your build command isn't producing output where Netlify expects it.
